## 23:00 — Agent SDK Engine End-to-End Test with LongCat Provider

使用 `~/.claude/longcat.settings.json` 的配置对 agent-sdk 引擎进行端到端测试，验证自定义 provider（LongCat）的完整链路。

### 测试过程

**环境准备：**
- 启动 `ainrf serve`（从 worktree 目录运行，确保加载最新代码）
- 生成测试 API key 并加入 `~/.ainrf/config.json`

**第一次任务（task_id: 89925ba8...）：**
- `api_key` 传递为 `ANTHROPIC_API_KEY`
- 结果：SDK 子进程启动成功，但 API 请求触发 10 次指数退保重试，最终失败
- 根因分析：`claude` CLI 使用 `x-api-key` 头部发送 `ANTHROPIC_API_KEY`，而 LongCat API 只接受 `Authorization: Bearer`（ANTHROPIC_AUTH_TOKEN），对 `x-api-key` 返回 401

**代码修复：**
- `agent_sdk.py` 的 `_provider_env()` 增加智能判断：
  - 当 `env_overrides` 中显式包含 `ANTHROPIC_AUTH_TOKEN` 时，不再设置 `ANTHROPIC_API_KEY`
  - 否则同时设置两者（向后兼容标准 Anthropic API）

**第二次任务（task_id: 11d38a13...）：**
- 在 `env_overrides` 中传入 `ANTHROPIC_AUTH_TOKEN`
- 遇到 Pydantic `extra="forbid"` 验证错误（worktree 代码未加载）

**服务重启从 worktree：**
- 之前 `uv run ainrf serve` 从主仓库运行，加载的是旧代码
- 改为从 worktree 目录启动后，Pydantic 验证通过

**第三次任务（task_id: ba4b6878...）：**
- 配置正确传递，引擎启动，session 创建（005a0e4a...）
- `ANTHROPIC_BASE_URL` 和 `ANTHROPIC_AUTH_TOKEN` 正确注入
- 但 claude CLI 在连接 LongCat API 时收到 `ConnectionRefused`，10 次重试后失败
- 直接命令行测试 `claude -p` 同样超时（120s timeout 仍 EXIT=124）
- curl 直接调用 LongCat `/v1/messages` 返回 200，说明 API 端点可用
- 结论：claude CLI（v2.1.133）与 LongCat API 之间存在底层连接兼容性问题，非引擎实现问题
