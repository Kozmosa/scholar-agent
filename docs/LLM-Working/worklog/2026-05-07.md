## 18:05 — 修复 flaky test `test_task_harness_remote_path_runs_without_readiness_precheck`

根因分析：
1. `fake_build_remote_launcher` mock 缺少 `ainrf_dir` 参数（生产代码签名已更新，测试未同步）
2. `fake_executor = object()` 没有 `close()` 方法，`build_remote_launcher` 将 `executor.close` 绑定为 `RunningProcess._cleanup`。任务完成后 `finally` 块调用 `process.cleanup()` → `executor.close()` 抛出 `AttributeError`，从 `finally` 块逃逸，覆盖了 `_stream_process_output` 的正常返回值，导致任务永远卡在 `running` 状态。

修复：
- 测试：给 `fake_executor` 添加 `close()` 方法；给 `fake_build_remote_launcher` 添加 `ainrf_dir` 参数
- 生产代码：`finally` 块中的 `await process.cleanup()` 用 try/except 包裹，防止 cleanup 错误掩盖任务结果

验证：完整后端测试套件 330 passed, 1 skipped（0 失败）

Commit: `8e0266b`
