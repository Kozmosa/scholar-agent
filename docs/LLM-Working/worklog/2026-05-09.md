### Critical Issues Found & Fixed

**1. `_FINAL_STATUSES` 缺少 `PAUSED` — `service.py:71`**
- 问题：`_FINAL_STATUSES` 只有 SUCCEEDED/FAILED/CANCELLED，不含 PAUSED
- 影响：服务重启时 PAUSED 任务被错误标记为 FAILED；cancel 时允许取消 PAUSED 任务
- 修复：将 `TaskHarnessStatus.PAUSED` 加入 `_FINAL_STATUSES`

**2. `pause_task()` 不更新数据库状态 — `service.py:599`**
- 问题：`pause_task` 只调用 `engine.pause(task_id)`，不更新 task 状态为 PAUSED
- 影响：任务在数据库中仍显示 RUNNING，resume 时状态检查失败，重启时被 kill
- 修复：在 `engine.pause()` 后添加 `self._update_task_status(task_id, status=TaskHarnessStatus.PAUSED)`

**3. `AgentSdkEngine.start()` 吞掉异常 — `agent_sdk.py:175`**
- 问题：`except Exception` 捕获所有异常并 emit error/task_failed 事件，但不 re-raise
- 影响：若 emit 本身失败，异常完全丢失，任务可能被错误标记为成功
- 修复：设置 `session.had_error = True`，然后 re-raise 异常，确保服务层的 `try/except` 总能捕获

**4. `_env_override` 全局 env 修改无文档 — `agent_sdk.py:43`**
- 问题：`_env_override` 修改 `os.environ` 是进程全局操作，缺乏文档说明
- 影响：未来开发者可能误以为可以并发运行多个 agent-sdk 任务
- 修复：在 docstring 中添加显式警告，说明 `_run_lock` 的串行化设计意图

### Important Issues Found & Fixed

**5. `get_engine()` 不验证实例类型 — `factory.py:13`**
- 修复：实例化后添加 `isinstance(instance, ExecutionEngine)` 检查，失败时抛出 `TypeError`

**6. `ResearchAgentProfileSnapshot` 类型缺失 — `frontend/src/types/index.ts:171`**
- 问题：前端 TypeScript 接口缺少后端新增的 6 个 provider 字段（api_base_url, api_key, default_opus_model, default_sonnet_model, default_haiku_model, env_overrides）及 model/permission_mode 等字段
- 修复：扩展接口定义，补全所有缺失字段

**7. `useMessageGroups` 内存泄漏 — `frontend/src/pages/tasks/useMessageGroups.ts:47`**
- 问题：`collapsedState` Set 只增不减，切换任务时旧 group IDs 残留
- 修复：`useEffect` 中同步时只保留当前 `grouped` 中存在的 ID，丢弃过期的

**8. `formatTime` locale 不一致 — `frontend/src/pages/tasks/MessageBlocks.tsx:5`**
- 问题：`toLocaleTimeString()` 无参数，依赖浏览器 locale，格式不一致
- 修复：固定为 `'zh-CN'` locale，统一 `hour12: false` 格式

## 01:00 — Agent SDK 引擎集成测试

补充 `tests/task_harness/test_agent_sdk_engine.py` 从 2 个基础测试扩展至 17 个覆盖全生命周期的集成测试。

### 新增测试覆盖

- **消息类型转换**：assistant text/thinking、tool_call/tool_result、user message、stream delta、rate_limit、system init
- **结果状态**：ResultMessage `is_error=True` 触发 `had_error` + `task_failed` 事件
- **生命周期控制**：`pause()` 设置 `should_pause_after_turn` → `start()`  emit `task_paused`；`resume()` 委托 `start()`
- **Prompt 队列**：`send_prompt()` 入队 → `start()` 优先消费 queued prompt
- **Abort 传播**：`abort()` 设置 abort_event → `start()` 中断循环并 raise `CancelledError`
- **异常处理**：`CancelledError` 直接 propagate；其他异常 emit error + task_failed 后 re-raise
- **Checkpoint 持久化**：session_id/turn_count/total_cost_usd 写入 JSON；启动时读取并恢复 session_id 和 pending_prompts
- **Env Override**：ANTHROPIC_API_KEY / ANTHROPIC_BASE_URL / ANTHROPIC_AUTH_TOKEN 正确注入；`env_overrides` 中显式设置 `ANTHROPIC_AUTH_TOKEN` 时跳过 `ANTHROPIC_API_KEY`
- **Hooks 注册**：验证 `PostToolUse` 和 `Notification` hook matcher 通过 options 传入 SDK

### 技术细节

- 使用 `pytest.mark.anyio` + `unittest.mock.patch` 隔离测试
- Fake SDK message types 通过 `patch` 替换引擎模块内的 `isinstance` 目标类，避免依赖真实 SDK 内部结构
- `_async_iter()` 辅助函数生成 async generator，使 `query` mock 可被 `async for` 正确消费
- `RaisingAsyncIter` / `BadAsyncIter` 自定义 async iterator 用于测试异常路径，避免 `async def` 返回 coroutine 导致的 `__aiter__` 类型错误
