关键设计决策

决策 选择 理由
TaskAttempt 与 Task 的关系 TaskAttempt 是独立表，关联现有 Task（task_id nullable） 不改 tasks 表结构，向后兼容；独立 session 操作不创建 Task 也行
Session 聚合字段 task_count / total_duration / total_cost 存在 TaskSession 表 避免每次列表查询都 JOIN 聚合，新 attempt 完成时更新
parent_attempt_id 自引用 FK → 形成 attempt chain Timeline 渲染需要按 chain 排列；可追溯完整的中断→修复→重试历史
token_usage_json JSON 列，结构：{"input": N, "output": N, "cache": N, "reasoning": N, "cost_usd": N} SQLite 支持 JSON 函数；Token Track 阶段直接读取聚合
Storage 沿用 SQLite，新增两张表在 task_harness DB 和现有 task 数据放一起，事务一致性
