Session Chain · Section 2/4

新增 Endpoints

Method Path 说明
GET /sessions 列表，支持 ?project_id=&status= 筛选
POST /sessions 创建 session（title, project_id）
GET /sessions/{id} Session 详情 + attempts 列表 + 聚合统计
PATCH /sessions/{id} 更新 title / status (archive)
DELETE /sessions/{id} Archive session（软删除）
GET /sessions/{id}/attempts Session 下的所有 attempts（按 seq 排序）

现有 Task API 的扩展

不改现有接口签名，只增加可选字段：

接口 新增字段 说明
POST /tasks session_id? (optional) 创建 task 时关联到 session
GET /tasks/{id} session_id, attempt_seq Response 中返回所属 session 信息
POST /tasks/{id}/prompt (via attempt chain) 发送 prompt 时自动在同 session 下创建新 attempt

TaskAttempt 创建时机（自动化）

Task Harness task 从 QUEUED → STARTING 时，自动创建 TaskAttempt
Task Harness task 完成/失败时，自动更新 TaskAttempt 的 status + duration + token_usage
Task Harness task 被 cancel 后 resume，或收到 prompt 后，创建新 attempt（parent=上一个）
API Route Session 聚合字段在每次 attempt 完成后自动 recalculation
