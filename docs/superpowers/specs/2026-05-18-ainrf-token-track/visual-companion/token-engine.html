AgentSdkEngine 改动 (agent_sdk.py)

_convert_sdk_message — ResultMessage 处理
当前 ResultMessage 分支（~line 324）只提取 session_id、num_turns、total_cost_usd。
新增提取：
sdk_msg.usage → {"input_tokens": N, "output_tokens": N, "cache_creation_input_tokens": N, "cache_read_input_tokens": N}
sdk_msg.model_usage → {"claude-opus-4-7": {同上}, ...}
sdk_msg.total_cost_usd → 已有，补充进 token_usage.total.cost_usd
构建 token_usage dict：
token_usage = {
  "total": {**usage, "cost_usd": total_cost_usd},
  "by_model": model_usage,
  "source": "agent-sdk"
}
然后：EngineEvent("system", "task_completed", payload, token_usage=token_usage)
另外：每个 AssistantMessage 也有 usage 字段（per-turn token 快照）→ 可作为 "token" 类型的 EngineEvent 实时发射

ClaudeCodeEngine 改动 (claude_code.py)

子进程退出后 → 轮询 session-meta
流程：
1. 在启动 Claude Code 子进程前，记录启动时间
2. 子进程退出后，轮询 ~/.claude/usage-data/session-meta/
3. 找到 start_time 最接近的 session-meta 文件（或通过 project_path + time 窗口匹配）
4. 最多等 30 秒（每秒检查一次新文件）
5. 提取 input_tokens + output_tokens
构建 token_usage dict（精简版）：
token_usage = {
  "total": {
    "input_tokens": meta["input_tokens"],
    "output_tokens": meta["output_tokens"]
  },
  "source": "claude-session-meta"
}
然后：EngineEvent("system", "task_completed", payload, token_usage=token_usage)
⚠ 超时未找到 session-meta → token_usage=None（静默跳过，不 block task 完成）
